# 意念情境 LINE Bot — 管理員使用手冊

> 給管理員（角色：admin）。你擁有全專案、全功能權限，且是**唯一能指派 admin 角色**的人。日常工作多在管理後台完成。

---

## 1. 登入後台

- 瀏覽器開：`https://linebot-admin.jiahd.cc/`，輸入帳號密碼（Basic Auth）登入。
- 兩個分頁：**Users**（人員）、**Projects**（專案）。

---

## 2. 新人上線（最常做的事）

每位新成員（員工 / 廠商 / 客戶）都照這三步：

1. 請對方先**加官方帳號好友**並**傳一則訊息** → 系統自動建檔（初始為 `visitor`）。
2. 到 **Users** 分頁找到這個人（可依角色篩選，新進通常在 `visitor`）。
3. 點「編輯」，設定：
   - **角色**（見下表）
   - **代號 alias**：小寫英數、**全系統唯一**、**設定後勿改**（工程看板就是用它點名收件人）
   - 若是 **vendor / customer**：勾選對方可看的**專案**

| 角色 | 可存取範圍 | 典型對象 |
|---|---|---|
| admin | 所有專案 + 可設別人為 admin | 公司負責人 |
| employee | 所有專案（不可設 admin） | 內部員工 |
| vendor | 僅被指派的專案 | 配合工班 |
| customer | 僅被指派的專案 | 屋主 / 客戶 |
| visitor | 無 | 尚未開通者 |

> 員工（employee）也能管理人員，但**不能**把任何人設成 admin。

---

## 3. 專案管理（Projects 分頁）

- **新增專案**：填名稱、選 Trello 看板、NAS 資料夾（預填案號建議值）→ 系統自動產生案號並在 NAS 建資料夾。
- **編輯**：可改名稱、Trello 看板、狀態（NAS 路徑唯讀）。
- **狀態語意**：
  - `active` 進行中（會跑通知）
  - `completed` 已完工（從 active 列表移出、**停止**通知）
  - `archived` 封存（停止通知，且 NAS 資料夾會搬到封存區）
- 封存共用 NAS 資料夾時會出現警告，請看清楚再確認。

---

## 4. 通知系統規則（你要懂，才能解釋給同事）

- 每天三班（早/中/下午）＋ 早上的**每日工程摘要**。
- **只有**帶 `[@(代號),日期區間]工項說明` 標記的項目才會觸發通知。
  ```
  [@(art),20260701-20260715]木作角料進場
  [@(冷leo)@(art),-20260730]冷氣定位驗收
  ```
- **完成定義**：卡片用「到期完成」、清單項目用打勾；完成後不再發到期/逾期提醒。
- 機器人答不出或涉及合約時，會 **escalate** 轉交通知：**目前未設定管理群組，會直接推給 `sa`（廖ㄚ莎）與 `larry`（詹阿瀨）兩人**。若要改推到一個管理 LINE 群組，需由 IT 設定 `LINE_NOTIFY_GROUP_ID`。

---

## 5. 廠商回報的追認流程

- 廠商在 LINE 按「標記完成」→ 工項**暫定**更新 + 你（及其他主管）收到一張**「待主管確認」卡片**。
- **✅ 確認** → 定案；**❌ 退回** → 看板還原成變更前狀態。
- 每次變更都會在該 Trello 卡片留稽核留言（誰、何時、做了什麼）。
- 早上摘要會列出目前所有「⏳ 待主管確認」項目，避免遺漏。

---

## 6. 維運注意事項

- ⚠️ **LINE 推播額度**：免費方案每月 200 則推播。用罄後**定時提醒與待確認卡片會送不出**（機器人回覆提問不受影響，因走 Reply API）。額度每月重置；要穩定發送需升級 LINE 方案。
- 技術部署（k8s / image 版本）由 **IT** 在 `jg-base` 維護，非後台操作範圍；遇到系統異常請找 IT。

---

## 7. 疑難排解

- **某人收不到通知？** 確認他：① 加好友 ② 角色正確 ③ 工項標記的代號＝他的 alias ④（vendor/customer）已被指派該專案。
- **代號衝突？** alias 全系統唯一，重複會被擋下並提示「此簡稱已被使用」。
- **舊案查不到？** 已 `archived` 的專案在 active 列表看不到，用「all」篩選即可。
