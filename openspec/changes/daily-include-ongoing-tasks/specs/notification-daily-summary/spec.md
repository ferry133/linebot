## ADDED Requirements

### Requirement: 進行中工項納入與呈現（執行中清單）
今日提醒的「進行中工項」段 SHALL 納入符合以下全部條件的工項：所屬清單名稱含「執行中」、**未完成**（card `dueComplete` 為否／checklist `state` 非 complete）、且帶有 `[@(alias)…]` 標記——**不論其開始/結束日期**（取代舊有 ±7 補完窗口判定）。凡納入且 `結束日 < 執行當日` 者 SHALL 標「逾期」紅字。

呈現規則：依看板分段、與上段急迫項（#1–#8）**去重**、無操作按鈕；card 層級工項（desc tag 未填 label，label 預設為卡片名）SHALL 以**卡片名**呈現，MUST NOT 因「label 等於卡片名且未逾期」而被略過。

對象：進行中工項段 SHALL 呈現給 **supervisor（`role ∈ {admin, employee}`，其可見範圍內全部）** 與 **該工項被 `[@(alias)]` 標記的 vendor（僅自己被指派者）**；與其無關者 MUST NOT 顯示。

#### Scenario: 執行中、未完成、日期不在 7 天內仍列入
- **WHEN** 工項在「執行中」清單、未完成、帶 tag，且結束日在 7 天以外（如 `70. [木工] 封板` `[@(木欽),20260626-20260715]`）
- **THEN** 該工項列入「進行中工項」段（不因窗口未開或日期遠而排除）

#### Scenario: card 層級無 label 以卡片名呈現
- **WHEN** 進行中工項為 card desc tag 且未填 label（label 預設為卡片名）、未逾期
- **THEN** 系統以卡片名呈現該工項，MUST NOT 略過

#### Scenario: 已完成 / 非執行中清單不列入
- **WHEN** 工項已完成，或其清單名稱不含「執行中」（如「未執行」「已完成」）
- **THEN** 該工項 MUST NOT 出現在「進行中工項」段

#### Scenario: 廠商看到自己的進行中工項
- **WHEN** role=vendor 收到/拉取每日內容，且有其被 tag 的執行中未完成工項
- **THEN** 內容含這些進行中工項；非其被指派者 MUST NOT 出現

#### Scenario: 逾期標記
- **WHEN** 進行中工項結束日已過（end < 今天）且未完成
- **THEN** 於該段標「逾期」紅字

#### Scenario: 與上段去重
- **WHEN** 某執行中工項同時在上段急迫區（#1–#8）出現
- **THEN** 下段進行中段 MUST NOT 重複列出該工項

## REMOVED Requirements

### Requirement: 摘要以補完窗口納入未完成工項並標記逾期
**Reason**: 「其餘進行中」段改為**清單導向**（執行中清單、未完成、不論日期），比 ±7 補完窗口更貼近「進行中」語意，且解決結束日 >7 天的進行中工項看不到的問題。
**Migration**: 由新要求「進行中工項納入與呈現（執行中清單）」取代——納入條件改為「執行中清單 + 未完成 + 有 tag」，逾期標記改以 `end < 今天` 判定；不再使用 ±7 補完窗口。
